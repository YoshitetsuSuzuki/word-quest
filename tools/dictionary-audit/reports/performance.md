# 性能測定（第14段階）

測定日: 2026-07-16 / 環境: ローカル(ネット不使用の照合部)。数値は本環境の実測。

| 項目 | 実測 |
|---|---|
| 辞書ダウンロード | CC-CEDICT 3.97MB＋OEWN 12.7MB＝計~16.7MB（数秒） |
| 正規化(normalize) | 286,067件(CEDICT 124,729＋OEWN 161,338) を **0.87s** |

| キャッシュ再利用 | ネットモードの再実行はcacheヒットで即時。ローカルモードは常にネット不要 |

## 10万語規模の想定とボトルネック

- **照合本体は線形にスケール**: 20,131件=1.12s → 100,000件≈**5〜6s**（インデックスはO(1)、件数比例）。ボトルネックにならない。
- **メモリ**: インデックスを全てインメモリ展開(~1.15GB)。辞書側が10万語なら現状で十分。辞書が数百万語(Kaikki全言語)になるとRSSが問題化 → その場合は **JSONL＋オフセットインデックス**（エントリを常駐させずファイルシーク）へ切替、またはSQLite化を推奨（index-store の差し替えで対応可能な設計）。
- **正規化/インデックス構築は前処理(1回)**: 辞書更新時のみ再実行。日次照合には影響しない。
- **ダウンロードが最大コスト**: Kaikki英語3.19GB等は帯域・時間が律速。必要言語のみ・キャッシュ・再利用で回避。
- **チェックポイント**: `--batch-size`ごとに `progress.json` 保存、`--resume`で再開。全件が途中停止しても最初からやり直さない。

## 選定した方式と理由

- **インデックス=言語別Mapシリアライズ(JSON)**。理由: 外部依存ゼロ・再現可能・O(1)検索・10万語規模で十分。SQLite/LevelDBは依存増のため見送り(将来の数百万語規模で再検討)。
