## 3. 判明した主要な問題パターン

1. **品詞タグ(tags)の系統的誤り（最多・約1,000件規模）**
   スキーマが `名詞/動詞/形容詞` の3種しかなく **副詞・前置詞・接続詞・代名詞・間投詞のタグが存在しない**。そのため副詞等が一律「名詞」等に押し込まれている。綴り・和訳・例文本体は正しく、tagsのみの問題。→ **スキーマ拡張＋一括再付与**が根本策。
2. **例文の語義ミスマッチ（High の中心・英語に集中）**
   見出し語の語義と例文の語義がずれる。特に固有名詞・商標への流用（`japan`漆→国名Japan、`latex`ゴム→組版LaTeX、`brad`鋲→人名Brad Pitt、`vat`大桶→付加価値税VAT 等）。
3. **例文欠落（データ欠損・英語で多数）**
   `example` が null の語が英語に散在（各バッチ10〜30件、報告のみ・誤りではない）。中国語は5件（zh-00426/声/式/越/咱）を明示。
4. **発音(IPA/ローマ字/拼音)** — 全件を外部発音辞書と照合していないため原則 not_verified。英語で異音語(heteronym)の疑い（live/close/use/read/lead に他品詞の発音）を確度中で提示。中国語の声調・韓国語のハングルには明白な誤り検出なし。
5. **⚠️ 要注意コンテンツ（ストア審査・4+レーティング観点で最重要）**
   - `geezer`(en-22829) 例文に差別的表現「slant-eyed」
   - `erection`(en-22571) 例文が不適切な別語義
   - `madman`(L6b) 和訳「気違い」＝差別語（「狂人」提案）
   - `since`(L1) の4択選択肢に不適切語が混入（監査対象外フィールドだが要確認）
   これらは辞書精度と別次元の**公開前必修正**項目。

## 4. 修正対応の区分

- **外部辞書未照合**: 全20,131件（`externalSourcesChecked: []`）。本監査は専門家の言語知識によるレビューであり、Wiktionary/CC-CEDICT/WordNet等への全件機械照合は実施していない（捏造回避のため正直に明示）。詳細は `dictionary-sources-and-licenses.md`。
- **自動修正可能候補（confidence≥0.98）**: 96件（別ファイル `proposed-fixes.json`、**未適用**）
  - 内訳: 品詞タグ64 / 例文語義24 / 大文字化3(独) / 例文欠落5(中) 他
  - **ただし真に機械適用が安全なのは独語の名詞大文字化3件のみ**（客観的・議論の余地なし）。残りは「品詞タグ＝スキーマ設計判断を伴う」「例文修正＝文章の作り直し（内容生成）」であり、確度が高くても**人手承認を要する**と判断。好みや言い換えでの上書きは行わない方針に従い、いずれも自動適用していない。
- **承認が必要な修正候補**: 1,603件（`audit-human-review.json`）

## 7. 残っているリスク

1. **発音(IPA等)の正確性**: 外部発音辞書未照合。異音語・音素欠落の疑いは提示したが確定には一次資料照合が必要（not_verified）。
2. **品詞タグ**: スキーマに副詞等がないため構造的に不正確。UI/機能に影響するため、修正は監査と切り離し別途設計判断が必要。
3. **例文欠落**: 英語に相当数。学習体験に影響しうるが、誤りではなくデータ欠損として報告のみ。
4. **ポルトガル語のブラジル基準**: 厳密なBR基準との一致は未確証（要人手）。
5. **要注意コンテンツ（§3-5）**: ストア公開前に必ず個別対応が必要。
