# 品詞タグ設計の調査（第4段階）

作成: 2026-07-15 / 本段階ではスキーマを変更しない（調査のみ）

## 現在利用可能なタグ（実データ実測）

`public/wordbank/**` 全20,131件を走査した結果、実在するタグは **5種のみ**：

| タグ | 件数 | 用途 |
|---|---|---|
| 名詞 | 9,311 | 品詞メタ |
| 動詞 | 2,992 | 品詞メタ |
| 形容詞 | 1,341 | 品詞メタ |
| 副詞 | 222 | 品詞メタ（中・韓のみ） |
| word | 6,265 | ピボット言語のプレースホルダ（品詞情報なし） |

`phrase`（表現セッション用の機能タグ）は語彙データには付与されず、別ファイル `phrases.json` 側で使用。

## 言語別の利用状況

| 言語 | 実在タグ | 副詞以下の品詞 |
|---|---|---|
| 英語 en | 名詞 / 動詞 / 形容詞 | **副詞すら無い**。代名詞・前置詞・接続詞・間投詞・限定詞・冠詞・助動詞・数詞をすべて名詞等へ押し込み |
| 中国語 zh | 名詞 / 動詞 / 形容詞 / 副詞 | 代名詞・介詞・接続詞・量詞・離合詞・数詞・間投詞が不足 |
| 韓国語 ko | 名詞 / 動詞 / 形容詞 / 副詞 | 代名詞・助詞・数詞・冠形詞・感嘆詞が不足 |
| 西/独/仏/葡/波/露 | word のみ | 品詞情報を一切保持しない |

これが監査で最多だった品詞タグ不一致（正規化後 **1,285件**）の構造的原因。特に英語は3タグしかなく、副詞（slightly, currently…）や代名詞（their, everyone…）が軒並み「名詞」になっている。

## UIや出題処理が依存しているタグ

コード全域（`src/`）を調査した結論：**品詞値（名詞/動詞/形容詞/副詞/word）を参照するロジックは存在しない。**

`tags` を読むのは以下のみ：
- `QuestionEngine.ts` … `tags.includes('phrase')` で表現問題を語彙問題から分離
- `QuestionEngine.ts` / `PhraseCardModal.tsx` … `tags.find(t => t !== 'phrase')` で**phrase問題のテーマラベル**を取得（tags[1]）
- `LocalQuestionRepository.ts` … `phrases.json` を `tags:['phrase',...]` で取り込み

選択肢（`choices`）は**各問題に静的に保存**され、実行時は `shuffle` するだけ。ピボット言語の動的ダミー生成（`localizedChoices`）も「その言語の訳を持つ語プール」から選ぶのみで、**品詞は一切見ていない**。品詞を画面表示する箇所も皆無（`.tsx` に品詞ラベル表示なし）。

## 不足している品詞

ご指定の16品詞について、現行スキーマでの表現可否：

| 品詞 | 表現可否（現状） |
|---|---|
| noun 名詞 / verb 動詞 / adjective 形容詞 | ✅ 可 |
| adverb 副詞 | △ 中・韓のみ可、**英語不可** |
| pronoun 代名詞 / preposition 前置詞 / conjunction 接続詞 / interjection 間投詞 | ❌ 全言語で不可 |
| determiner 限定詞 / article 冠詞 / auxiliary 助動詞 / numeral 数詞 | ❌ 不可 |
| particle 助詞 / classifier 量詞（中・韓で必要） | ❌ 不可 |
| phrase | ✅ 既存機能タグ |
| other | ❌（未定義） |

## 既存タグ追加による互換性リスク

**実質ゼロ。** 品詞タグはアプリの `phrase` 判定・テーマ以外のロジックから参照されないため、`副詞` `代名詞` `前置詞` 等を追加しても、出題・選択肢・スコア・UI・学習ロジックのいずれも影響を受けない。

留意点（守るべき不変条件）：
1. `phrase` タグの意味・位置（表現問題の識別子）を変えない。
2. phrase問題の `tags[1]`（テーマ）構造を壊さない。
3. `tags` を空配列や未定義にしない（`q.tags?.includes` はnull安全だが、既存の非空前提を維持）。
4. 日本語ラベル（名詞…）と英語canonical（noun…）のどちらを保存値にするかを決め、全言語で統一する（現状は日本語ラベル＋pivotは "word"）。

## データ移行が必要な件数

- 監査が明示的に指摘した品詞タグ誤り：**1,285件**（en 848 / zh 289 / ko 140 / es 4 / fr 2 / de 1 / ru 1）。
- ただし**真の移行対象はこれより多い可能性が高い**。英語は副詞タグが存在しないため、監査で個別指摘されなかった副詞・代名詞・前置詞も含めれば全英語8,136件の再タグ付けが視野に入る。ピボット6言語（6,265件）は全て "word" 固定で品詞情報ゼロ。
- **移行は本監査と切り離し、外部辞書照合（カテゴリB）を前提に別途実施すべき**。品詞確定をAI判断のみで行わない（ルール5）。提案とマッピング候補は `proposed-pos-migration.json`（未適用）に出力済み。

## 推奨アプローチ（提案・未実施）

1. スキーマにcanonicalな品詞集合（`proposed-pos-migration.json` の16種）を定義。保存値の表記（日本語 or 英語）を確定。
2. 機能に影響しないため、**まず英語の副詞・代名詞・前置詞・接続詞・間投詞から**外部辞書（Wiktionary/WordNet等）照合で再タグ付け。
3. 中韓の代名詞・介詞・量詞・数詞を追加照合。
4. ピボット言語は各言語辞書で品詞を付与（優先度低・機能影響なし）。
5. `phrase` タグ運用は現状維持。
