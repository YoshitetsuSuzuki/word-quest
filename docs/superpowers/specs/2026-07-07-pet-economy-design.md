# 相棒キャラの経済・レア度設計

**Goal:** 「勉強→通貨→キャラ獲得」のバランスを整え、コインでは買えない伝説キャラを含むコレクション性を作る。

## 現状の獲得レート（実測・2026-07-07）
- 1問正解 = baseCoin 5 × 難易度倍率(1.0〜2.8) × コンボ倍率(1.0〜3.0) ≒ 8〜15コイン/問
- 1日の収入: 軽め(デイリー10問＋ログイン＋ミッション1) ≒ 150〜200 / しっかり(20〜25問＋ミッション満了＋バトル/レイド) ≒ 700〜900
- 節目ボーナス(別枠): 7日+300, 30日+1,500, 100日+8,000 等

## 通貨（2階建て）
- **コイン**（ソフト通貨・既存）: 勉強で貯まる。コモン〜レアキャラの購入に使う。
- **ジェム**（ハード通貨・新規 `User.gems`）: 希少。伝説キャラ専用。
  - 入手: 節目/実績でごく少量、イベント、(ネイティブ化後)リアル課金。
  - 配布たたき台: ストリーク 7→+2 / 14→+3 / 30→+8 / 50→+15 / 100→+40 / 365→+150。主要実績 +1〜3。

## レア度と価格（確定たたき台）
| レア度 | 種(例) | 解放 |
|---|---|---|
| Common(無料) | 草 / 炎 / 水 | 開始時に1体選択 |
| Common(コイン) | 上記の他種・土・風 | **1,200コイン**/体 |
| Rare | 光 / 闇 / 雷 | **4,000コイン** または 実績「相棒をLv50に」 |
| Legendary | 虹(プリズム) / 星 | **ジェム限定(例30)** または 高難度実績(100日ストリーク/相棒Lv100) |

- 所有上限 `PET_MAX_PETS` を 3→**6** に拡張（コレクション性）。
- 学習XPは従来どおり**アクティブ1体だけ**。

## データ構造
- `User.gems: number`（migrate/default=0）。
- キャラ図鑑 `petCatalog.ts`: `{ id(species), rarity, cost:{coin?|gem?}, unlockAchievement? }[]`。
- `User.ownedSpecies: PetSpeciesId[]`（解放済み種）。pets[] は「育成中の個体」、ownedSpecies は「解放済みの種類」。
  - 新規個体を追加する時、解放済み種から選ぶ or 未解放種をその場で購入して解放。

## UI
- ヘッダーにジェム表示（コインの隣）。
- 相棒スイッチャーの「＋」→ **キャラ図鑑モーダル**（種一覧・レア度色・価格・ロック状態・解放ボタン）。
- 解放条件を満たさない伝説には「🔒 ジェム30」「🔒 100日ストリーク」等を表示。

## 実装フェーズ
1. **経済基盤**: gems通貨(型/default/migrate/ヘッダー表示)＋ジェム配布(ストリーク/実績)＋コモン価格を1,200へ。
2. **新種の追加**: PetSprite に 光/闇/雷/虹 の配色・モチーフを追加（フォーム別シルエットは共通流用）。
3. **キャラ図鑑UI**: petCatalog＋解放フロー＋スイッチャー刷新。

## テスト
- 価格/解放判定(コイン不足・ジェム不足・実績未達で不可、達成で解放)。
- ジェム配布が節目で正しく入る。migrate後方互換(gems/ownedSpecies既定)。
