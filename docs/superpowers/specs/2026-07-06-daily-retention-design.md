# デイリーリテンション機構(案A: ストリーク背骨型) 設計書

日付: 2026-07-06 / 承認: よしてつ様(案A採用。案C育成キャラ型は将来検討)

## 目的

毎日アプリを開くモチベーションを作る。動機4タイプ(損失回避・報酬期待・成長可視化・新鮮さ)を
「🔥学習ストリーク」を背骨に統合する。コアループは**3分(10問)**。通知は使わない(アプリ内完結)。

## 制約

- localStorage完結・サーバーレス・Pay to Winなし(既存方針)
- 数値・報酬は `src/data/*.config.ts` のデータ駆動(コード改変なしで調整可)
- `featureFlags` でON/OFF可能
- 既存のログインボーナス・ミッション・レイドは壊さない(上に載せる)

## コンポーネント

### 1. 学習ストリーク(損失回避・中核)

- **定義**: 「1日に10問以上回答した日」の連続日数。ログインだけでは継続しない。
- User拡張: `studyStreak` / `longestStudyStreak` / `lastStudyDate` / `streakFreezes` / `claimedStreakMilestones`
- 判定: その日の回答数が閾値(config: 10)に達した瞬間に判定。
  前回スタンプ日との差が1日→+1、2日かつフリーズ保有→フリーズ消費で+1、それ以外→1にリセット。
- **節目報酬**(config駆動): 3/7/14/30/50/100/365日 → コイン+限定称号(ショップの称号として付与)。
  受取済みは `claimedStreakMilestones` で管理、Celebration演出。
- **フリーズ**: ショップに消耗品「ストリークフリーズ」(500コイン、最大2個ストック)。
  1日休んでも自動消費で途切れない。コインの使い道=プレイ動機にもなる。
- **喪失警告**: 今日のスタンプ未獲得かつ streak>=3 のとき、ホームに
  「🔥{streak}日のストリークが今夜消えます」を表示。

### 2. デイリーループカード(ホーム最上部)

3ステップの「今日の一式」をカード表示。全て完了で✨演出+カード縮小。

1. 📝 今日の10問(todayAnswered/10)
2. 📖 今日の単語を見る(下記3)
3. 🎁 ログインボーナス(既存・自動受取済み表示)

### 3. 今日の単語(報酬期待+新鮮さ)

- 日付シードで決定的に1語選出(現在カテゴリのverified語から `hash(dateStr) % pool`)。
- タップで開く: 単語・発音記号・訳・例文・🔊読み上げ。`todayWordSeenDate` で既読管理。
- サーバー不要、全ユーザー同日同語(カテゴリ毎)。

### 4. ログボ7日カレンダー(報酬期待)

- 既存 applyLoginBonus を7日サイクルの見えるカレンダーUIに強化。
- LoginBonusModal に7マス表示、当日ハイライト、7日目は大きな報酬(config)。
- サイクル位置は `(streakDays - 1) % 7`(既存streakDays=ログイン連続を流用)。

### 5. 週間学習グラフ+図鑑バー(成長可視化)

- User拡張: `dailyHistory: Record<string, number>`(日付→回答数、14日分でプルーニング)。
- ホームに直近7日の棒グラフ(小型・CSSのみ)。
- まなびタブに級別の習得プログレスバー(習得語数/級の語数)。

### 6. 日替わりチャレンジ(新鮮さ)

- 日付シードから1件生成(例: 「リスニングで5問正解」「Lv3だけ10問」「コンボ5を出す」)。
- 既存デイリーミッション枠に「⭐日替わり」として追加表示、報酬は通常より高め。

## データフロー

- 回答時: `GameContext.answerQuestion` → todayAnswered更新(既存) → 閾値到達でストリーク判定+
  dailyHistory更新 → 節目ならCelebration。
- 起動時: `migrate()` で新フィールド補完(既存ユーザーの streakDays は学習ストリーク初期値に引き継ぐ)。
- 日付判定は既存 `todayStr()`(dateUtils)に統一。

## エラー処理・エッジ

- 端末日付の巻き戻し: lastStudyDate が未来なら何もしない(ストリーク維持)。
- NaN/欠損: migrate で数値フィールドを Number.isFinite ガード(既存パターン踏襲)。
- フリーズ自動消費は「2日ぶり回答」時のみ。3日以上はリセット(フリーズは消費しない)。

## テスト

- ストリーク判定ロジックは純関数 `src/core/StreakEngine.ts` に切り出し、
  日付差・フリーズ・リセット・節目の各ケースを node での机上テストで確認。
- UIはプレビューで実機検証(スタンプ獲得→ストリーク+1→節目演出→フリーズ購入→翌々日回復)。

## 実装フェーズ

1. StreakEngine+User拡張+migrate+ホームのストリーク表示/警告(中核)
2. デイリーループカード+今日の単語
3. ログボカレンダー+週間グラフ+図鑑バー
4. 日替わりチャレンジ+ショップのフリーズ+節目称号

## 対象外(YAGNI)

- プッシュ通知(将来ストア版でローカル通知)
- シーズンランキング(案B)・育成キャラ(案C、**後日検討と約束済み**)
- ストリークランキングへの反映
